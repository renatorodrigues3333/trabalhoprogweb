Guia Prático — Frontend

Projeto ArenaPlay (trabalhoprogweb) · Como configurar estilos, layouts e campos da tela

1. O projeto tem DOIS sistemas de estilo

Entender isto evita 90% da confusão. Cada tipo de página usa um sistema diferente:

SistemaOnde é usadoComo o estilo é aplicado
Bootstrap 5 + CSS próprio Páginas públicas e painéis (welcome, arenas, dashboards) que usam @extends('layouts.main') Classes do Bootstrap (container, btn, card...) + classes próprias em public/css/style.css
Tailwind + componentes Jetstream Telas de autenticação (login, register, registerArenaOwners) que usam <x-guest-layout> Classes do Tailwind (mt-4, w-full...) direto no HTML + componentes <x-input>, <x-button>
Regra de ouro: olhe a primeira linha da view. Se começa com @extends('layouts.main') → é Bootstrap. Se começa com <x-guest-layout> ou <x-app-layout> → é Tailwind.

4. Como funciona o LAYOUT base (Bootstrap)

O layout é o "molde" da página. Em layouts/main.blade.php existe a navbar fixa e um ponto onde o conteúdo de cada página é injetado:

<!-- layouts/main.blade.php (resumido) -->
<body>
    <nav> ... menu ... </nav>
    <main>
        @yield('content')   <!-- AQUI entra o conteudo de cada pagina -->
    </main>
</body>

Cada página "preenche" esse molde:

<!-- resources/views/Arenas/index.blade.php -->
@extends('layouts.main')          {{-- usa o molde --}}
@section('title', 'Minhas Arenas') {{-- preenche o @yield('title') --}}

@section('content')                {{-- preenche o @yield('content') --}}
    <h1>Minhas Arenas</h1>
@endsection
No layoutNa páginaFunção
@yield('content')@section('content') ... @endsectionBloco grande de conteúdo
@yield('title')@section('title', 'Texto')Valor curto (título da aba)
Para mudar a navbar de todas as páginas de uma vez, edite layouts/main.blade.php — a alteração reflete em toda página que usa @extends('layouts.main').

5. Componentes Blade (telas de auth — Tailwind)

As telas de login/cadastro usam componentes reutilizáveis (começam com x-). Eles ficam em resources/views/components/. Em vez de escrever um <input> com 10 classes, você escreve:

<x-label for="email" value="Email" />
<x-input id="email" type="email" name="email" class="block mt-1 w-full" required />
<x-button>Salvar</x-button>
ComponenteViraArquivo
<x-input />Um <input> estilizadocomponents/input.blade.php
<x-label />Uma <label>components/label.blade.php
<x-button>...</x-button>Um <button> escurocomponents/button.blade.php
<x-validation-errors />Lista de erros de validaçãocomponents/validation-errors.blade.php

Customizar a aparência de um componente

Para mudar o estilo de TODOS os inputs de auth de uma vez, edite o componente. Ex.: components/input.blade.php:

<input {!! $attributes->merge([
    'class' => 'border-gray-300 focus:border-indigo-500 rounded-md shadow-sm'
]) !!}>

Troque as classes Tailwind dentro de 'class' => '...'. O $attributes->merge junta essas classes-padrão com qualquer class="..." que você passar na hora de usar o componente.

Tailwind x Bootstrap: no Tailwind, o estilo é montado por muitas classes pequenas no próprio HTML (px-4 py-2 bg-gray-800 rounded-md). No Bootstrap, você usa classes prontas maiores (btn btn-primary). Não misture os dois na mesma tela.

6. RECEITA: adicionar um novo campo a um formulário

Adicionar um campo na tela exige tocar em até 4 lugares (de cima pra baixo no fluxo). Exemplo: adicionar o campo "site" numa arena.

  1. View (o campo na tela) — adicione o input no formulário: 
    <div class="mb-3">
    <input type="text" name="site" class="form-control"
           placeholder="Site da arena">
</div>
  2. Controller (validar + salvar) — inclua o campo na validação e no create: 
    $validated = $request->validate([
    // ... campos existentes ...
    'site' => ['nullable', 'url', 'max:255'],
]);
Arena::create([ /* ... */ 'website' => $validated['site'] ?? null ]);
  3. Model (permitir salvar) — adicione a coluna ao $fillable: 
    protected $fillable = [ /* ... */ 'website' ];
  4. Migration (criar a coluna no banco) — crie e rode uma migration: 
    php artisan make:migration add_website_to_arenas_table

// dentro do up():
$table->string('website', 255)->nullable();

php artisan migrate
Cuidado com os nomes! O name="site" do input é o que o controller lê em $request. A coluna no banco (website) é o que vai no $fillable e no create. Eles podem ter nomes diferentes — só não confunda qual usar em cada lugar.

7. RECEITA: exibir dados na tela

Para mostrar dados que o controller enviou (ex.: a lista de arenas em $arenas):

@foreach($arenas as $arena)
    <div class="card mb-3">
        <div class="card-body">
            <h5>{{ $arena->name }}</h5>
            <p>{{ $arena->address }}</p>
        </div>
    </div>
@endforeach
Quero...Uso
Imprimir um valor{{ $arena->name }}
Repetir para cada item@foreach($lista as $item) ... @endforeach
Mostrar só se houver dados@if($arenas->count()) ... @else Nenhuma arena @endif
Valor padrão se vazio{{ $arena->phone ?? 'Sem telefone' }}
Link para outra página<a href="{{ route('arenas.edit', $arena->id) }}">
Lembre-se: as colunas no banco são em inglês (name, address, phone). Usar $arena->nome mostra vazio. Use sempre o nome real da coluna.

8. Checklist rápido de frontend

  1. É página pública? Use @extends('layouts.main') + classes Bootstrap.
  2. É tela de auth? Use <x-guest-layout> + componentes x- + Tailwind.
  3. Quer mudar cor/estilo global das páginas públicas? Edite public/css/style.css.
  4. Quer mudar a navbar de todas as páginas? Edite layouts/main.blade.php.
  5. Novo campo no form? View → Controller (validar) → Model ($fillable) → Migration.
  6. Todo formulário POST precisa de @csrf logo após a tag <form>.
  7. Para exibir dados, use o nome real da coluna (inglês), não o do formulário.
